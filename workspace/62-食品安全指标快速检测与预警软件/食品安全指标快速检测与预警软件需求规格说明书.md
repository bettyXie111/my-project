# 食品安全指标快速检测与预警软件需求规格说明书

## 行业背景

食品安全指标快速检测与预警软件面向食品检测中心、采样岗位、检测岗位、复核岗位与管理岗位，用于把样本接收、检测录入、结果复核、阈值预警与审计留痕串成可追踪闭环。
在实际业务中，样本来源、批次编号、检测指标、复检结论和处置动作常分散在纸质单据、Excel 表格和消息沟通中，容易出现口径不一致、状态不同步、风险发现滞后与归档困难的问题。
系统通过样本台账、检测任务、预警中心、统计报表和审计留痕五类核心能力，支撑值班人员在单一控制台中完成样本流转与风险处置。

## 用户角色

- 采样人员：登记样本来源、批次号、采样时间与流转状态，发起样本入库或移交。
- 检测人员：接收检测任务、录入检测指标、提交结果并触发复核。
- 审核人员：复核异常样本、确认超标结论、给出处置意见。
- 管理员：维护基础字典、指标阈值、菜单权限、报表配置与审计留痕。

## 技术栈

- 前端框架/运行时：React + Vite + TypeScript。
- 后端框架/运行时：FastAPI + Python 3.11。
- 数据库/存储：SQLite（本地交付）/ PostgreSQL（生产可选）。
- 认证/身份方案：JWT + RBAC（本地账号体系）。
- 部署目标：Windows 本地运行，支持 Docker 作为可选交付方式。
- CI/CD 工具：GitHub Actions（如无则暂不配置）。
- 测试栈：pytest（后端）+ Playwright（可选 E2E）。

## 核心业务流程

1. 用户登录后进入首页，查看样本总量、待复核任务、异常预警与今日处理进度。
2. 采样人员在样本管理中登记样本来源、编号、批次、检测对象与流转状态。
3. 检测人员在工作台创建或接收检测任务，录入指标结果并保存留痕。
4. 系统根据指标阈值与复核规则生成预警，提示审核人员处理超标、缺项或复检异常。
5. 审核人员查看样本详情、任务详情与历史记录，完成查看、编辑、复核和删除确认等操作。
6. 管理员在流程配置中维护阈值、菜单权限、模板规则与审计范围。
7. 系统生成统计报表与审计日志，支持按时间、指标、批次、来源单位和状态筛选导出。

异常与边界：
- 样本编号重复时必须阻止提交，并提示重复来源。
- 阈值越界时必须进入预警状态，并保留确认人和确认时间。
- 删除操作必须二次确认，避免误删已复核样本。

## 数据模型

核心实体与关系（简化）：
- Sample(样本)：sample_id, sample_code, source, batch_no, status, created_at。
- DetectionTask(检测任务)：task_id, task_name, sample_id, detect_mode, current_step, created_at。
- DetectionResult(检测结果)：result_id, task_id, indicator_name, indicator_value, threshold, conclusion。
- AlertRecord(预警记录)：alert_id, sample_id, level, reason, handled_by, handled_at。
- WorkflowTemplate(流程模板)：template_id, template_name, step_count, active_flag。
- AuditLog(审计日志)：actor_id, action, target, detail_json, ts。
- User/Role/Permission：用于 JWT + RBAC 权限控制。

数据范围：
- 采样人员仅能查看本人或本班次样本。
- 检测人员可查看分配给自己的任务与结果记录。
- 审核人员可查看预警样本与处置历史。
- 管理员可查看全部数据和审计日志。

## 功能列表

### 样本管理
- 样本登记、新增、编辑、删除、查看、导出。
- 样本编号、来源单位、批次号、检测对象与状态筛选。

### 检测工作台
- 任务创建、任务分配、任务状态跟踪、结果录入。
- 支持新增、编辑、查看、删除等操作入口与弹窗确认。

### 预警中心
- 阈值预警、异常样本提醒、复核确认、处置闭环。
- 支持风险分级、过滤和导出。

### 统计报表
- 按批次、指标、来源单位、状态统计趋势与导出。
- 提供列表和汇总两种视图。

### 审计留痕
- 记录登录、提交、复核、删除与导出等关键动作。
- 便于后续追溯样本处理链路。

## 边界约束

- 首版优先保证样本流转、检测录入、预警确认和导出留痕四条主链路。
- 不引入复杂外部系统实时对接，避免首版耦合过重。
- 状态变更必须有明确操作人和时间戳。
- 预警阈值和流程模板由管理员统一维护，普通用户只可查看。
- 接口采用 REST 风格，返回 JSON。

## 验收标准

1. 登录后可看到样本管理、工作台、预警中心、统计报表与审计留痕入口。
2. 能完成“样本登记→任务创建→结果录入→预警确认→报表导出”的闭环。
3. 新增、编辑、查看、删除等操作均有可见入口，并能触发确认弹窗。
4. 样本状态、预警状态与审计记录可按条件检索，且导出结果可打开。
5. 页面与接口无报错，且至少一组自动化测试通过。

## 视觉风格关键词

- 深色工业控制台风格，配色以深蓝/青绿为主，突出风险提醒色。
- 布局强调左侧导航 + 右侧内容区，信息密度高但分层清晰。
- 按钮、标签、状态卡片采用统一圆角与统一层级，避免模板化 SaaS 风格。
- 适合在桌面端操作，登录页、首页与业务页具有明显差异。

## 说明

本 PRD 聚焦样本接收、检测任务、预警处置与审计留痕，功能边界与当前 Web 系统实现保持一致。
