# 技术选型表

本文用于把当前项目的 PRD 技术要求、现有实现状态和可选扩展统一对齐，便于后续按“需求目录 wrapper + 公共骨架引擎”的方式继续演进。

## 选型结论

| 技术项 | 已实现 | 待实现 | 可选 | 当前项目对应说明 |
|---|---|---|---|---|
| 前端框架/运行时 |  | React + Vite + TypeScript |  | 当前 `apps/web/index.html` 仍是静态单页演示骨架，未落到 `package.json`、`vite.config.ts`、`tsx` 工程。 |
| 后端框架/运行时 | FastAPI + Python 3.x |  |  | `apps/api/main.py` 已以 `FastAPI` 作为入口，属于已落地部分。 |
| 数据库/存储 | SQLite 文件 | PostgreSQL |  | 当前 `apps/api/services/store.py` 直接使用 `sqlite3` 文件存储，符合本地交付。 |
| 认证/身份方案 |  | JWT + RBAC |  | 当前仅有前端演示登录，没有真正的令牌、角色、权限控制。 |
| 测试栈 | 后端基础冒烟测试（`unittest`） | pytest + pytest-asyncio | Playwright E2E | 当前有 `apps/api/tests/test_health.py`，但还未切到 PRD 建议测试栈。 |
| 部署目标 | Windows 本地运行 | Docker |  | 当前材料生成与演示链路以本地 Windows 运行最稳。 |
| CI/CD 工具 |  | GitHub Actions |  | 当前仓库未落 CI 配置，适合后续再补。 |
| API 设计 | REST + JSON |  |  | 当前 `apps/api/main.py` 和路由层已按 REST 风格组织。 |
| 权限边界 |  | 角色级访问控制 |  | 当前没有后端鉴权与权限过滤，后续如果扩展多角色需要补齐。 |
| 数据库迁移 |  | Alembic |  | 当前 SQLite 文件直连，未引入迁移层。若未来增加 PostgreSQL，再补迁移。 |

## 当前项目建议

- **保留实现**：`FastAPI + Python + SQLite 文件 + REST + Windows 本地运行`
- **优先补齐**：`React + Vite + TypeScript`、`JWT + RBAC`、`pytest`
- **按需扩展**：`Docker`、`PostgreSQL`、`Playwright`、`GitHub Actions`

## 文件级对照

- `apps/api/main.py`：后端入口，已实现 `FastAPI`
- `apps/api/services/store.py`：数据库访问，已实现 `SQLite 文件`
- `apps/api/tests/test_health.py`：基础测试，待迁移到 `pytest`
- `apps/web/index.html`：前端演示骨架，待迁移到 `React + Vite + TypeScript`
- `食品安全指标快速检测与预警软件需求规格说明书.md`：PRD 技术栈定义来源

## 迁移顺序建议

1. 先把前端演示骨架替换成 `React + Vite + TypeScript`
2. 再补 `JWT + RBAC`
3. 将测试从 `unittest` 迁到 `pytest + pytest-asyncio`
4. 如果后续需要多实例或部署环境，补 `Docker` 和 `PostgreSQL`
5. 最后再接入 `Playwright` 做 E2E
