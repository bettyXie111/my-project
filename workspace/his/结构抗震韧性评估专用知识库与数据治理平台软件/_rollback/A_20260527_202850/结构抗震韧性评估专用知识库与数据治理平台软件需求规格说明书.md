# 结构抗震韧性评估专用知识库与数据治理平台软件需求规格说明书

版本：V1.0  
日期：2026-05-27  

## 1. 行业背景

结构抗震韧性评估涉及多源数据（结构信息、震害记录、场地条件、监测数据、构件清单、加固方案、评估模型参数等）的收集、清洗、校核、归档与复用。实际工作中常见问题包括：数据口径不一致、指标计算链路不可追溯、评估依据与结论难以复核、历史工程知识难以沉淀为可检索的“可用知识”。  

本软件面向“结构抗震韧性评估”场景，提供两类核心能力：

1) 专用知识库：沉淀工程项目、结构对象、评估指标、模型与案例结论，支持检索、关联与复用。  
2) 数据治理：对数据集的来源、版本、质量规则、校核记录、权限与血缘进行管理，确保评估结果可追溯、可复核、可审计。  

## 2. 用户角色

| 角色 | 典型职责 | 主要诉求 |
|---|---|---|
| 评估工程师 | 建立项目与结构对象、配置指标、提交评估并形成结论 | 快速建模、标准化指标、可复核的评估链路 |
| 数据治理专员 | 维护数据集、口径、质量规则、版本与血缘 | 数据可用性与一致性、质量可量化、可审计 |
| 审核/复核人员 | 对评估过程与结果进行复核、抽检、追溯 | 能看到数据来源、计算依据、变更记录 |
| 项目管理员 | 用户与权限管理、系统字典维护、导出归档 | 权限清晰、审计完整、对外交付方便 |

## 3. 核心业务流程

### 3.1 项目建库与结构对象建模
1. 管理员创建“评估项目”，填写项目基础信息（区域、重要性等级、场地类别等）。  
2. 评估工程师在项目下创建“结构对象”（建筑/桥梁/构筑物），维护结构类型、层数、材料体系、建成年代、设计抗震设防信息等。  
3. 系统为每个结构对象生成唯一编号，作为后续数据集关联与评估记录的主键。  

### 3.2 数据集登记、版本与质量规则
1. 数据治理专员登记数据集（如：构件清单、材料检测、现场调查、监测时程、震害样本等），填写来源与责任人。  
2. 为数据集建立版本（V1、V2…），每次变更需填写变更说明与影响范围。  
3. 为数据集配置质量规则（完整性、范围、唯一性、值域、枚举、关联一致性），执行质量检查并记录结果。  
4. 系统记录数据血缘：数据集版本 →（清洗/汇总/计算）→ 指标/评估记录。  

### 3.3 指标配置与评估执行
1. 评估工程师在结构对象下选择指标模板或自定义指标（例如：承载能力裕度、构件延性指标、层间位移角、功能恢复时间、关键构件破坏概率等）。  
2. 为指标选择数据来源（数据集版本）与计算方法（公式/脚本/人工录入），并声明关键假设。  
3. 提交评估任务，系统计算或记录指标值，形成“评估记录”，自动生成可追溯链路（指标 → 数据集版本 → 质量检查记录）。  

### 3.4 结果复核与知识沉淀
1. 复核人员查看评估记录，检查：数据来源是否正确、质量是否达标、假设是否合理、结论是否与证据一致。  
2. 通过/驳回复核：驳回需注明原因并退回重算或补充数据。  
3. 将“评估结论”与“关键证据”归档为“案例知识条目”，支持按结构类型/指标/区域/震级等条件检索。  

## 4. 数据模型

### 4.1 主要实体

| 实体 | 字段（示例） | 说明 |
|---|---|---|
| Project（项目） | id, name, region, site_class, importance, owner, created_at | 评估工作的顶层容器 |
| Structure（结构对象） | id, project_id, name, type, year_built, stories, material_system, fortification_intensity | 项目下的评估对象 |
| Dataset（数据集） | id, name, category, owner, source_desc, sensitivity_level | 数据资源登记 |
| DatasetVersion（数据集版本） | id, dataset_id, version, change_note, created_at | 版本化管理 |
| QualityRule（质量规则） | id, dataset_id, rule_type, rule_expr, severity | 数据质量规则 |
| QualityRun（质量执行记录） | id, dataset_version_id, status, score, detail_json, run_at | 质量检查结果 |
| Indicator（指标） | id, structure_id, name, method, unit, data_bindings_json | 指标配置 |
| Assessment（评估记录） | id, structure_id, status, summary, reviewer, reviewed_at | 评估任务与结论 |
| LineageEdge（血缘边） | id, from_type, from_id, to_type, to_id, note | 追溯链路 |
| KnowledgeEntry（知识条目） | id, title, tags, evidence_refs_json, created_at | 专用知识库条目 |
| AuditLog（审计日志） | id, actor, action, object_type, object_id, detail, at | 关键操作留痕 |

### 4.2 关键关系
- `Project 1..n Structure`  
- `Dataset 1..n DatasetVersion`  
- `Dataset 1..n QualityRule`  
- `DatasetVersion 1..n QualityRun`  
- `Structure 1..n Indicator`  
- `Structure 1..n Assessment`  
- `Indicator n..1 DatasetVersion`（通过 bindings 绑定）  
- `LineageEdge` 连接数据与指标/评估，支撑可追溯性  

## 5. 功能列表

### 5.1 登录与权限
- 用户登录（内置账号，初始管理员：admin/admin123）。  
- 角色：管理员、工程师、治理专员、复核员。  
- RBAC：按模块与数据敏感级别控制可见与可操作范围。  

### 5.2 项目与结构管理（知识库骨架）
- 项目：新增/编辑/查看/删除、列表筛选。  
- 结构对象：新增/编辑/查看/删除、关联项目。  
- 结构对象详情页：展示指标、评估记录、关联的数据集与血缘。  

### 5.3 数据集治理
- 数据集登记：新增/编辑/查看/删除。  
- 数据集版本：新增版本、查看变更说明、版本对比（字段级差异以简化文本展示）。  
- 质量规则：配置规则类型（必填、值域、唯一性、枚举、关联一致性），支持启用/禁用。  
- 质量检查：对指定版本执行规则，输出评分与问题列表。  
- 血缘图：展示“数据集版本 → 指标 → 评估记录”的链路（以列表/简图两种视图）。  

### 5.4 指标与评估
- 指标库：按结构对象维护指标，支持从模板创建。  
- 评估执行：创建评估任务、关联指标集合、生成评估摘要。  
- 复核：通过/驳回，记录原因与时间。  

### 5.5 知识条目
- 从评估结论“一键沉淀”为知识条目：包含结论摘要、关键指标、证据引用（数据集版本、质量记录、图表说明）。  
- 检索：按标签、结构类型、区域、指标关键词检索。  

### 5.6 审计与导出
- 审计日志查询（按时间、对象类型、操作者筛选）。  
- 导出：项目概览、评估记录、质量检查报告（导出为 JSON/Markdown 下载）。  

## 6. 边界约束

- 本版本不实现复杂的数值仿真与大型有限元求解；指标计算以“配置化公式 + 示例脚本/人工录入”方式实现。  
- 不接入外部真实监测系统；数据上传以 JSON/CSV（本版本仅实现 JSON）示例为主。  
- 不实现多租户与组织级隔离；以单组织内部权限控制为主。  
- 不实现复杂图数据库；血缘以关系表形式表达，并提供简化可视化。  

## 7. 验收标准

### 7.1 功能验收
- 能完成：项目→结构对象→数据集版本→质量规则/检查→指标绑定→评估记录→复核→沉淀知识条目 的闭环。  
- 关键页面可访问且无前端报错：登录、首页、项目、结构对象、数据集、质量检查、指标、评估、知识库、审计。  
- API 提供基础 CRUD 与关键动作接口（执行质量检查、提交评估、复核、沉淀知识）。  

### 7.2 数据治理验收
- 质量规则执行结果可追溯到具体版本，并保留问题明细。  
- 血缘链路能从评估记录追溯到数据集版本与质量检查记录。  

### 7.3 非功能验收
- 自动化验证通过：包含至少 1 组后端单元测试（TestClient）与总行数门禁（>=3000 行非空代码行）。  
- 生成软著三件套：操作手册、源代码文档、申请表的 md/docx/pdf 产物齐全；截图 >= 12 张且引用有效。  

## 8. 视觉风格关键词

视觉主题：**“工程蓝图 + 抗震韧性图谱”**  

关键词：
- 深海蓝 / 灰白底（工程制图感）
- 细网格背景与分区卡片（图纸/坐标纸意象）
- 指标与血缘采用“节点/连线”视觉语言（简化图谱）
- 关键状态使用对比色：通过（青绿）、风险（橙）、失败（红）
- 表格与表单偏“工程台账”排版，强调字段对齐与可复核性

建议字体：系统默认中文字体（宋体/微软雅黑混排均可），标题加粗、正文克制。
