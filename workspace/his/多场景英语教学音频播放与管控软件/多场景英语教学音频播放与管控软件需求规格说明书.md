# 多场景英语教学音频播放与管控软件 需求规格说明书

版本：V1.0  
日期：2026-05-28  

## 1. 行业背景
在校内外英语教学中，音频材料通常分散在教师个人设备、U 盘或各类网盘中，课堂播放依赖单机播放器与临时连接音箱，难以做到“同一套资源在不同教学场景下稳定播放”。当教学场景切换（普通教室 / 语音室 / 口语角 / 线上直播 / 自习室）时，播放控制方式、设备能力与课堂节奏差异明显，常见问题包括：找音频耗时、播放中断、音量突变、学生端与教师端不同步、无法快速定位句子片段、无法统一管控学生端播放权限等。

本软件面向“多场景英语教学”，把音频资源、课堂流程与播放控制合并到一个可管控的平台：同一份课件音频可以在不同场景下按规则播放；教师能快速发起播放、暂停、片段循环、倍速等；管理员可配置设备与权限，减少课堂临时操作。

## 2. 用户角色
1) 英语教师  
- 需要按课时与活动快速定位音频；需要一键播放/暂停/片段循环；需要在课堂中对学生端播放进行管控；需要课堂结束后复盘播放记录。  

2) 学生  
- 在语音室或自习室需要跟随教师播放；在自学模式下可按权限回听；需要清晰的播放进度与提示。  

3) 教务管理员/信息化管理员  
- 需要配置场景、教室设备、播放终端；需要管理账号、角色权限；需要查看系统运行日志与异常。  

## 3. 核心业务流程
### 3.1 课前：资源与课时准备
1) 教师创建“课时”(Lesson)，选择教材单元、目标场景（普通教室/语音室/线上直播/口语角/自习室）。  
2) 教师为课时绑定音频素材：可按“段落/句子”切片，设置片段起止时间、循环次数建议、默认倍速。  
3) 管理员配置场景能力：例如语音室允许学生端强制同步，线上直播禁止学生端自主跳转，自习室允许学生端自由回听。  

### 3.2 课中：播放与管控
1) 教师进入“课堂控制台”，选择课时并“开始上课”。  
2) 教师选择片段点击播放，系统在教师端播放，同时向学生端下发控制指令（按场景规则决定是否强制同步）。  
3) 教师可执行：暂停/继续、跳转片段、A-B 循环、倍速、音量建议、全体静音提示（学生端只显示提示不直接改系统音量）。  
4) 系统记录每次控制事件，形成“课堂播放日志”。  

### 3.3 课后：复盘与作业化
1) 教师查看课时播放记录：播放次数、常回听片段、异常中断。  
2) 教师可把片段发布为“跟读任务”（仅在自习室/学生自学可见），并设置有效期与允许回听次数。  
3) 管理员查看设备在线率与异常日志，定位网络或播放器问题。  

## 4. 数据模型
### 4.1 用户与权限
- User：id，name，role(teacher/student/admin)，password_hash，created_at  
- RolePolicy：role，scene_capabilities（每个场景下的能力开关集合）  

### 4.2 场景与设备
- Scene：id，name（普通教室/语音室/线上直播/口语角/自习室），capabilities（同步/跳转/倍速/循环等）  
- Device：id，scene_id，name，type(teacher_console/student_terminal/speaker_bridge)，online，last_seen  

### 4.3 课程与音频
- Lesson：id，title，book_unit，scene_id，teacher_id，status(draft/running/ended)  
- AudioAsset：id，title，duration_sec，source_uri（演示版用本地静态路径模拟）  
- Clip：id，lesson_id，audio_asset_id，label，start_ms，end_ms，default_speed，loop_suggest  

### 4.4 控制与日志
- PlaybackSession：id，lesson_id，started_at，ended_at  
- ControlEvent：id，session_id，actor_user_id，target（teacher/all_students/single_device），action(play/pause/seek/ab_loop/speed)，payload，created_at  

## 5. 功能列表
### 5.1 账号与登录
- 账号登录（演示版：内置 teacher/admin/student 测试账号）。  
- 基于角色的页面入口控制。  

### 5.2 场景配置
- 场景能力开关：是否强制同步、是否允许学生跳转、是否允许倍速、是否允许片段循环。  
- 设备列表：模拟在线/离线状态与最后心跳时间。  

### 5.3 课时与片段管理
- 新建课时、选择场景、绑定音频素材。  
- 片段切分：设置起止时间、标签、默认倍速与循环建议。  

### 5.4 课堂控制台
- 播放控制：播放/暂停/跳转片段、A-B 循环、倍速。  
- 管控策略：根据场景能力决定学生端是否跟随。  
- 课堂日志：控制事件时间线。  

### 5.5 学生端播放页
- 显示当前片段与进度（演示版以事件流模拟）。  
- 根据场景策略限制跳转与倍速。  

### 5.6 统计与复盘
- 单课时：播放次数、常用片段、异常次数（演示版基于事件日志统计）。  
- 设备统计：在线率、异常事件。  

## 6. 边界约束
- 演示版不对接真实音频文件下载与 DRM；音频资源以静态路径/模拟时长方式展示。  
- 演示版不做真实 WebSocket 多端同步，使用“轮询 + 事件日志”方式模拟学生端跟随。  
- 不涉及学生隐私数据采集；仅记录课堂控制事件。  
- 不对系统音量做强制修改；仅提供“音量建议”提示。  

## 7. 验收标准
1) 系统可启动：API 服务正常运行，前端页面可访问。  
2) 关键页面：登录页、教师控制台、学生端页面、场景/设备管理页至少可用且无报错。  
3) 核心流程：教师创建课时 → 选择片段播放 → 生成控制事件 → 学生端可看到事件并按规则限制。  
4) 场景差异：至少 3 个场景策略可切换并影响学生端权限（例如语音室强制同步、线上直播禁止学生跳转、自习室允许回听）。  
5) 自动化验证：至少 1 组 API 冒烟测试通过；总代码非空行数 >= 3000。  
6) 文档一致：操作手册、申请表、源代码文档与系统功能一致，截图 >= 8 张。  

## 8. 视觉风格关键词
关键词：教学控制台、清爽蓝白、卡片化、时间轴日志、设备面板、轻量图标、可读性优先。  
落地要求：  
- 主色调采用偏冷的蓝色系，背景浅灰/白，强调信息层级。  
- 教师控制台突出“片段列表 + 当前播放状态 + 控制按钮”。  
- 日志采用时间轴样式，事件条目简洁，不使用花哨动效。
