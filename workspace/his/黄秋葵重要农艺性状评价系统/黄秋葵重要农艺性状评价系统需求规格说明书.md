# 黄秋葵重要农艺性状评价系统 需求规格说明书

## 行业背景
黄秋葵（Okra）在不同生态区、栽培季节与管理方式下的产量与品质波动明显。育种试验与品种比较试验通常需要同时跟踪多个农艺性状（株高、始花期、单株结荚数、商品荚比例、产量构成、病害发生等级等），并在多点、多季、多重复条件下进行统计分析与排名。传统的 Excel 方式容易出现：性状口径不统一、重复录入、缺失值处理随意、评分规则不透明、试验对比结论难复核等问题。

本系统面向“黄秋葵品种（或材料）重要农艺性状”评价场景，提供从试验设计、数据采集、质量核验、性状标准化、综合评分、对比分析到报告导出的闭环，目标是让“评分规则可配置、数据过程可追溯、结论可复核”。

## 用户角色
- 试验负责人：创建试验、定义性状与评分规则、发布数据填报任务、审核结果、导出报告。
- 调查员/记录员：按小区（plot）或植株（plant）录入观测数据，补充照片与备注。
- 数据管理员：维护品种库、性状口径库、病虫害字典、量表与单位，处理历史数据导入与清洗。
- 统计分析人员：对比不同试验、筛选材料、查看综合评分分解，导出用于论文/简报的数据表。

## 核心业务流程
1. 建库准备：数据管理员维护品种（材料）档案、性状字典（名称/单位/测定方法/允许范围）、评分模板（权重、方向性、区间映射）。
2. 新建试验：试验负责人录入试验地点、季节、管理措施、重复次数；选择参试品种与小区布局（区组/随机/自定义）。
3. 数据采集：调查员以“小区-日期-性状”为单位填报观测值；系统即时校验范围、单位与缺失；支持批量导入。
4. 质量核验：试验负责人查看异常提示（超范围、同一小区同日重复、缺失过多、量纲不一致），可退回或批准。
5. 性状标准化：系统按评分模板将原始观测值转换为标准分（0~100），并对缺失值按规则处理（不计、均值、最低分等可选）。
6. 综合评价：按权重汇总得到综合评分；支持按“季节/地点/管理措施”分层比较，生成材料排名与分解雷达图数据。
7. 结论输出：导出试验报告（含评分口径、异常处理记录、排名表、性状分解表），保证可复核。

## 数据模型
### 1) 品种/材料（Variety）
- id：唯一标识
- name：材料名称
- source：来源/育种单位
- type：品种/杂交组合/系谱材料
- remark：备注

### 2) 试验（Trial）
- id
- name：试验名称（可含季节与地点）
- location：地点
- season：季节/批次
- design：试验设计（随机区组/裂区/自定义）
- replicates：重复数
- manager：负责人
- created_at

### 3) 小区（Plot）
- id
- trial_id
- block：区组号
- code：小区编号
- variety_id
- area_m2：面积
- management_tags：管理措施标签（滴灌/露地/覆膜等）

### 4) 性状字典（Trait）
- id
- code：性状编码（如 PH、DF、YPP）
- name：性状名称（株高/始花期/单株结荚数等）
- unit：单位（cm、d、个、kg/亩等）
- direction：越大越好/越小越好/目标区间最佳
- method：测定方法与口径
- min_value/max_value：合理范围（用于校验）

### 5) 观测记录（Observation）
- id
- plot_id
- trait_id
- observed_at：观测日期
- value：数值
- note：备注
- operator：录入人
- status：草稿/已提交/已批准/已退回

### 6) 评分模板（ScoringProfile）
- id
- name
- items：每个 trait 的权重、标准化方式、缺失处理方式
- version：便于试验锁定口径

### 7) 评分结果（Score）
- trial_id
- variety_id
- trait_scores：各性状标准分
- total_score：综合评分
- explain：缺失/异常处理摘要

## 功能列表
### A. 基础数据
- 品种库管理：新增/编辑/禁用；支持导入 CSV/JSON。
- 性状字典管理：单位、方向性、范围、测定方法；支持口径版本化。
- 病虫害与等级字典：0~9 级或自定义量表。
- 评分模板：按性状配置权重与标准化方式；可复制模板并调整版本。

### B. 试验管理
- 创建试验：地点、季节、设计、重复；选择参试品种。
- 小区布局：区组、编号、材料分配；支持导入布局表。
- 数据任务：生成填报清单（按日期或阶段），设置截止时间。

### C. 数据采集与校验
- 观测录入：按试验→小区→性状录入；支持批量粘贴与文件导入。
- 即时校验：范围校验、单位提示、同日重复提示、缺失提示。
- 审核流转：提交/退回/批准；记录操作日志。

### D. 评价与分析
- 性状标准化：min-max、z-score、分段映射、目标区间惩罚等。
- 缺失处理：不计入、按区组均值、按全试验均值、按最低分等策略。
- 综合评分：权重汇总；支持按分层条件生成多套排名。
- 结果解释：展示每个材料的得分分解与“拉低项/拉升项”。

### E. 报告与导出
- 导出：排名表、原始数据表、标准分表、异常处理日志。
- 报告：包含试验信息、评分口径、统计摘要、结论建议（可编辑）。

## 边界约束
- 本系统聚焦“黄秋葵农艺性状评价”，不覆盖完整农场生产管理（采购、库存、财务）。
- 统计分析以常用的标准化与加权评分为主，不承诺替代专业统计软件的全部能力。
- 账户与权限采用轻量角色模型；对接统一身份认证属于扩展项。
- 数据存储默认本地 SQLite（演示/单机），生产可替换为 Postgres（扩展项）。

## 验收标准
- PRD 字段与口径完整：包含行业背景/用户角色/核心业务流程/数据模型/功能列表/边界约束/验收标准/视觉风格关键词。
- 试验闭环可演示：能创建试验→导入小区布局→录入观测→审核→生成评分与排名→导出报表。
- 校验可用：录入超范围、重复、缺失过多可触发提示并在异常日志中可见。
- 评分可复核：评分模板与标准化参数可导出；相同原始数据重复计算结果一致。
- 关键页面与 API 无报错：启动后首页与主要业务页可访问，至少 1 组接口/页面冒烟测试通过。
- 代码规模满足门禁：`apps/` 与 `packages/` 的非空代码行数 ≥ 3000。

## 视觉风格关键词
- 主题：田间试验记录、手账式研究笔记
- 色彩：深绿 + 黄秋葵荚果黄绿点缀（低饱和）
- 字体与排版：信息密度中等，表格清晰，模块标题像“试验卡片”
- 图形元素：细线网格、标尺刻度、标签贴纸样式（但不花哨）
- 交互感受：强调“口径与追溯”，每个结果页都有“评分口径/异常处理”入口
